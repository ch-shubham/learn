## Basic Intro

Java is a high-level, class-based, object-oriented programming language designed to have as few implementation dependencies as possible. It is widely used for building web applications, mobile applications, and enterprise software. Java is platform-independent at the bytecode level, meaning you can write code once and run it anywhere with a Java Virtual Machine (JVM).


## Type Converstion and Type Casting

```java
int num = 100;
double result = num; // Implicit conversion

double num = 100.5;
int result = (int) num; // Explicit casting -> Output is 100

byte a = 10;

```

- **Widening** -> Two datatypes are automatically converted. This happens when we assign value of smaller datatype to bigger datatype. And two datatypes must be compatible.
- **Narrowing** -> This happens when we forcefully do the conversion like float to int etc. (Overflow issue could happen here, so it is risky sometimes)

## Difference between int and Integer (Similarly between others)

- `int` is primitive and `Integer` is reference or Object type.
- Many primitive types have wrapper classes like int have Integer.
- All the primitive datatypes can be represented in the Object form. It is helpful to access helper methods.

## Pass By Value and Reference

Java is always pass-by-value. Primitive: Copies the value. Object/Non-Primitive: Copies the reference. The object's state can be modified. The caller's reference itself cannot be changed.

## Scope

- There are multiple scopes namely: Method Scope, Block Scope, Loop Scope
- **Shadowing** - Shadowing occurs when a local variable has the same name as an instance variable. (Not recommended)

## Variable arguments (Varargs)

- Varargs allow passing a variable number of arguments to a method. Internally, they are treated as an array.

## Collections In Java

- **Collection** and Collections are different things in java. A Collection is an interface, whereas Collections is a class. Collections can hold only Object and can not hold primitive data types like int, double etc.
- **Set** -> Unique. Ex: `Set fruits = new HashSet();`
- **List** -> Ordered, duplicates. Ex: `List<String> fruits = new ArrayList();`
- **Queue** -> FIFO. Ex: `Queue<Integer> fruits = new LinkedList();`
- **Map** -> Key `<->` Value. `putIfAbsent` can be used instead of `put` to be sure that insertion happens if there is no already key present. So we can avoid bugs and error of overwriting the element. `Map<String, Integer> fruitsCalories = new HashMap();`
- **Arrays** -> `int [] arrayName = new int[size];` and `int [][] my2DArray = new int[rows][columns];`. It is good practice to provide the size of array at compile time and NEVER at runtime because the input size from user can be ANYTHING which is very dangerous. Array -> fixed size, primitives supported. ArrayList -> dynamic size, Objects only (use Integer instead of int).

## Strings

- A String in Java is a object that represents the collection or sequence of characters. It is represented by the java.lang.String class. String is not a primitive data type. It is a final class data type which provides several helper functions. By default Strings are Immutable. We can not directly change the characters of string as they are immutable, but we can change the string as a whole if needed. So to make the string mutable we make use of StringBuilder. To calculate the length of string we have .length() method and not `.length`. `String s = "AnyString";` If you’re repeatedly building a string inside a loop, use StringBuilder. Prefer StringBuilder over String += inside loops

## Iterators

Iterator is an object used to loop through elements of collections. The iterator() method in ArrayList returns an Iterator type object. Iterator method is provided by collection interface and since MAP is not a part of the collection interface iterator is not available on map. Access to iterator is by : Set, List, Queue. With map we can achieve the same thing but with different technique. We use entrySet() in map for this.
Methods provided by iterators are:

- hasNext(): Checks if there are more elements to iterate. Returns true if more elements are available; otherwise, false.
- next(): Retrieves the next element. Always call hasNext() before next() to avoid a NoSuchElementException.
- remove(): Deletes the last element returned by next(). This method can only be used once after calling next().
- forEachRemaining(Consumer\<? super E> action): A Java 8 method that applies a given action to all remaining elements. It automates the iteration process, so you don’t need to manually check for more elements.

```java
// ---------- Iterator ----------
Iterator<Integer> it = list.iterator();
it.hasNext();                // More elements?
it.next();                   // Get next element
it.remove();                 // Remove current element safely
// ---------- Typical Usage ----------
Iterator<Integer> it = list.iterator();
while (it.hasNext()) {
    int x = it.next();
    if (x % 2 == 0) {
        it.remove();         // Safe removal while iterating
    }
}
```

### Choose the Right Data Structure

| Need                     | Use                              |
| ------------------------ | -------------------------------- |
| O(1) lookup              | `HashMap`, `HashSet`             |
| Sorted data              | `TreeMap`, `TreeSet`             |
| Min/Max repeatedly       | `PriorityQueue`                  |
| FIFO                     | `Queue (ArrayDeque)`             |
| LIFO                     | `Deque (ArrayDeque)`             |
| Random access            | `ArrayList`                      |
| Frequency counting       | `HashMap`                        |
| Preserve insertion order | `LinkedHashMap`, `LinkedHashSet` |

### Bit Manipulation

```java
n << k;                 // Left shift (n * 2^k)
n >> k;                 // Right shift with sign (n / 2^k)
n >>> k;                // Unsigned right shift (fills with 0s)
n & mask;               // AND (check/extract bits)
n | mask;               // OR (set bits)
n ^ mask;               // XOR (toggle bits / find unique)
~n;                     // NOT (invert all bits)
(n & 1) == 0;           // Is even?
(n & 1) == 1;           // Is odd?
n & (n - 1);            // Clear lowest set bit
(n & (n - 1)) == 0;     // Is power of 2? (n > 0)
n & -n;                 // Isolate lowest set bit (Fenwick Tree)
n |= (1 << i);          // Set i-th bit
n &= ~(1 << i);         // Clear i-th bit
n ^= (1 << i);          // Toggle i-th bit
(n & (1 << i)) != 0;    // Is i-th bit set?
Integer.bitCount(n);                    // Count set bits
Integer.numberOfLeadingZeros(n);        // Leading zeros
Integer.highestOneBit(n);               // Highest power of 2 <= n
Integer.lowestOneBit(n);                // Lowest set bit
```
